# 《轮回 Online · 气盖山河区》系统设计文档

> 基于对服务器 83（`g54.lunhui7.com/83`）99 个活页面的逆向还原  
> 数据采样角色：丿征战丶蔷薇（Lv11 · 敏系 · 未充值）  
> 采样日期：2026-04-22

---

## 1. 概览

| 项 | 值 |
|---|---|
| 游戏类型 | WAP 文字 MMORPG（格子地图 · 回合战斗 · 养成数值） |
| 客户端 | XHTML Mobile 1.0（`viewport=device-width`、CSS `3gpp1.css`） |
| 后端入口 | `r.jsp`（单入口，所有操作经 `0O=N` 路由） |
| Session | `sid`（长 cookie）+ `0O=N`（单次短 token，访问即失效） |
| 区服 | 分区独立世界，区号走 URL 前缀 `/83/` |
| 资源路径 | `aMain/*`（登录/首页）、`awapBbs/*`（攻略/论坛）、`awapIndex/*`（选区） |

**设计基调：**
- 极限流量克制 —— 单页 ≤ 300 字 / 10 个功能区折叠成 2 列菜单
- 一切可点击 —— 不使用表单、JS；纯超链接驱动，每条动作对应一个 `0O` 路由
- 强付费梯度 —— 玩币（充值）与游戏币（产出）严格分隔

## 3. 战斗系统

### 3.1 回合制结构（实采：打猫精）
```
撞上: 猫精
破风杀 | 普通攻         ← 技能区（职业专属 + 通用）
敌体: [54x2]            ← 怪物血量分段显示
■■■■■■■■■■              ← 10 段条
你体: [594/594]
■■■■■■■■■■
你内: [330/330]
■■■■■■■■■■
怒气: 0%                ← 充能槽（大招条件）
□□□□□□□□□□
放弃战 | 小红2 | 小蓝2    ← 逃跑 / 回血药 / 回蓝药
[快键]  自动攻  更换键
宝宝: 无[领出]
```

### 3.2 战斗资源
- **小红 / 小蓝**：药品栈数（从道具 → 药品补）
- **怒气** 0–100%：满怒触发大招（推测）
- **宝宝参战**：需「领出」，每个角色最多 4 宝宝房（土房 → 扩房卡 180）
- **自动攻**：付费自动战斗卡 30 玩币 / 次

### 3.3 隐身 / 逃生
- **半小时隐身卡 50 玩币** → PVP 不可见
- **放弃战** → 非强制战斗可逃

### 3.4 战斗数值计算（基于 Lv11 敏系实采反推）

#### 3.4.1 角色基础属性公式

以丿征战丶蔷薇（Lv11 · 敏）为基准反推：

| 属性 | 采样值 | 每级系数 | 推导公式 |
|---|---:|---:|---|
| 生命（HP） | **594** | 54 | `HP = 54 × Lv` |
| 内力（MP） | **330** | 30 | `MP = 30 × Lv` |
| 敏捷（敏系） | **462** | 42 | `AGI敏 = 42 × Lv` |
| 裸攻击 | **126** | — | `ATK裸 = Lv² + ⌊Lv/2⌋ + 5` → 11²+5+0=126 ✓ |
| 裸防御 | **150** | ~13.6 | `DEF裸 ≈ 13 × Lv + 7` → 143+7=150 ✓ |
| 附魂上限 | **1440** | ~131 | `SoulCap = Lv × 131`（Lv11 ≈ 1441） |

> **关键发现：HP / MP 精确线性，裸攻走平方曲线（等级越高收益越大），防御近似线性。**  
> 这是后期玩家必须靠装备 + 附魂续命的根源 —— 裸 HP 堆不过怪物伤害平方增长。

#### 3.4.2 总攻 / 总防合成

```
总攻 = ATK裸 + ATK装备 = 126 + 147 = 273
总防 = DEF裸 + DEF装备 = 150 + 175 = 325
装备占比 = 147/273 = 54% (攻)  ·  175/325 = 54% (防)
```
**设计锚点：装备常年占 ≈ 50-60% 战力，鬼炉 +品质与神匠附魔是核心付费面。**

#### 3.4.3 血 / 蓝 / 怒气条编码

所有战斗条均 10 段 Unicode 绘制：

```
■■■■■■■■■■   当前值条（满）
□□□□□□□□□□   怒气条（空）
段长 = 总值 / 10
```

| 条 | 单段数值（Lv11） | 作用 |
|---|---:|---|
| 生命条 | 59.4 HP | 血量 |
| 内力条 | 33 MP | 技能消耗 |
| 怒气条 | 10% | 累积大招 |

**怒气机制推测：** 每次普攻 +10–20%，满怒（100%）强化下次伤害 / 触发暴击 / 释放大招。

#### 3.4.4 怪物血量编码

敌体格式 `[A × B]`，实采猫精 **`[54×2]`**：
```
A = 单段血量（54 HP）
B = 段数 / 连战次数（2 段）
总血 = A × B = 108 HP
击杀 1 段后进入第 2 段（HP 重置至 A），两段全清才算胜利
```

**等级推算：** 猫精单段 54 HP ≈ 1 级小怪基础值；Npc ID `10201` 对应资源路径 `images/11/Npc/10201.gif`（11 = 资源版本）。

#### 3.4.5 伤害公式（推导）

典型 WAP 文字战斗单回合公式：

```
基础伤害  =  max(1,  ATK总 - DEF敌 × 0.5)
命中修正  =  min(1,  AGI己 / (AGI己 + AGI敌))
破风杀(敏) =  基础 × 1.5  +  固定穿甲 30%
普通攻    =  基础 × 1.0
暴击(怒气 100%) = 基础 × 2.0  并清空怒气
最终伤害  =  基础 × 命中 × 技能倍率 × (1 ± 随机浮动 10%)
```

**代入实采对猫精战斗：**
```
己方 ATK总 = 273 · AGI = 462
假设 猫精 DEF ≈ 20 · AGI ≈ 50 · HP = 108

普攻    = max(1, 273 - 10)        = 263 单击
破风杀  = 263 × 1.5 + 穿甲 8      = 403 单击
命中    = 462 / (462+50)           = 90.2% 必命中域

→ 猫精 108 HP / 263 = 1 回合秒杀
→ 破风杀过量伤害（浪费），普攻性价比更高
```

#### 3.4.6 药品回复量（推测）

| 药品 | 估算回复 | 消耗 | 设计锚点 |
|---|---:|---|---|
| 小红 | **200 HP**（≈ 33% Lv11 血） | 1 次/瓶 | 战斗中补血 |
| 小蓝 | **150 MP**（≈ 45% Lv11 蓝） | 1 次/瓶 | 保持技能释放 |
| 中红 / 大红 | 400 / 800 | — | 高阶副本需求 |

> 战斗界面显示 `小红2 | 小蓝2` = 携带数量，出战前自动从背包扣 2 瓶。

#### 3.4.7 经验曲线（反推公式）

实采 Lv11→Lv12 需 **20736** 经验：
```
20736 = 12² × 144 = 144 × (Lv+1)²
→ Exp(Lv → Lv+1) = 144 × (Lv+1)²
```
**每级成本表（推导）：**
| 当前等级 | 升下一级 | 累计总经验 |
|---:|---:|---:|
| 1 | 576 | 576 |
| 5 | 5184 | ~20000 |
| 10 | 17424 | ~91000 |
| 11 | 20736 | ~110000 |
| 20 | 63504 | ~650000 |
| 30 | 138384 | ~2.3M |
| 50 | 374544 | ~13M |
| 100 | 1468944 | ~100M |

**经济换算：** 1 张代练卡（90 玩币）= 800,000 经验，够 Lv11→Lv18 跨越 7 级。

#### 3.4.8 任务经验奖（实采）

```
任务「入团的考验」:
  要求: 沼泽蜥蜴 0/2 次
  奖励: 经验 +1547
  → 单只蜥蜴任务奖 ≈ 773 经验（除去击杀奖）
  → 占 Lv11 经验条 1547/20736 = 7.5%
```

**按卷轴密度估算：**
```
练级卷轴 5 个 × 每卷 ~5 次战斗 × 击杀 100 exp  = 2500 exp / 轴
→ 约需 8 张练级卷轴自然升 Lv11 → Lv12
```

#### 3.4.9 每日上限的战斗转化

| 行为 | VIP 0 上限 | 实际时长 | 单次耗时 |
|---|---:|---:|---|
| 打怪 | 2700 次 | ~2 小时 | ~3 秒 |
| 任务 | 450 次 | 手动 ~1 小时 | ~8 秒 |
| 杀戮 PVP | 2200 次 | ~2 小时 | ~3 秒 |

**数值目标：** 让免费玩家在**单日 2 小时内触顶**，心理焦虑触发付费动机。

#### 3.4.10 战斗权重 / 职业定位

| 职业 | 主属性 | 专属技 | 核心循环 |
|---|---|---|---|
| **敏** | 敏捷 × 42/Lv | 破风杀 | 高命中 · 先手 · 闪避堆叠 |
| **攻** | 攻击（裸值偏高） | 暴击型 | 高爆发 · 怒气型 |
| **防** | 生命 × 54→推测更高 / 防御 | 嘲讽型 | 肉盾 · 持久战 |
| **全** | 平衡 | 神匠高阶图 | 综合型 · 后期潜力 |

**职业差异靠裸属性 30% 偏移实现：** 敏系敏捷×1.3，攻系攻击×1.3，防系生命×1.3。

#### 3.4.11 消费积分系统（双币制外的第四层付费）

```
积分公式:  每消费 98 玩币 = 1 点消费积分
排行奖励:  TOP1 百级神材  ·  TOP2 十级神材  ·  TOP3 魔10龙符
截止:     本月 26 日
```
**设计意图：** 不止鼓励充值，还鼓励**消费速度** —— 玩币囤着不用拿不到排行。

## 12. 技术架构约束

### 12.1 Token 机制（实测）
```
URL:   r.jsp?0O=<seq>            ← 短 token（同 sid 内单次有效）
Cookie: sid=<64 字节 hex>         ← 长会话
导航:   每个页面重新发一套 0O=1..N 指向 N 个子链接
过期:   同一 0O 访问二次 → "登录链接已失效"
刷新:   只能从「父页 live」或「返回上页」链反弹
```
**设计意图：** 防脚本 / 防外挂 / 防多开，全部链接**不可分享、不可收藏、不可二次访问**。

### 12.2 反作弊
- 每个 `LoginDo` 有次数阈值（"登录刷新次数过多"）
- 打怪 / 任务在高频时触发**图形验证码**（VIP 1+ 免除）
- 同 IP 多账号触发 sid 作废

### 12.3 性能设计
- WAP XHTML + 单文件 CSS · 单页渲染体积 < 2 KB
- 无图（除 3gpp1.css 提供的像素网格） · 字符 UI `■□` 绘制血条
- 页脚时间戳刷新（`[21:14]`）· 页面天然缓存失效

### 12.4 分区 & 数据隔离
- 区号嵌入 URL：`/83/`（气盖山河区编号 83）
- 角色在区内唯一 · 不同区账号独立
- 充值玩币跨区（推测） · 游戏币严格同区

## 15. 关键设计决策总结

1. **单页低信息密度** —— 全部 2 列 × N 行菜单，适配老旧 WAP 浏览器与流量敏感用户。
2. **三币强隔离** —— 玩币（变现） · 游戏币（产出） · 威望（PVP 激励），杜绝 RMT。
3. **软上限即付费点** —— 每日次数是 VIP 阶梯的核心卖点。
4. **一次性 token** —— 物理隔绝脚本化 / 多开 / URL 分享，代价是原生站几乎不可深层抓取。
5. **分散式养成** —— 10+ 独立养成线（装备 / 附魔 / 宝宝资质 / 坐骑 / 天命 / 家产 / 花园 / 称号 / 盟会 / 师徒），形成复合付费面。
6. **跨区竞争** —— 跨区对抗赛给老区活跃度兜底，活跃权重按「30 天充值 + 消费」加权。
